## Development Photos (`concept-images/`)

Drop photos into `assets/images/concept-images/` using these exact filenames. The dev cards on the main page will pick them up automatically — no code changes needed.

| Development | Card image (concept-images/) | Logo (logos/) |
|---|---|---|
| Gran Verona | `gran-verona.webp` ✓ | `gran-verona-logo.png` ✓ |
| Playa Clara | `playa-clara.webp` ✓ | `playa-clara-logo.svg` ✓ |
| Playar | `playar.webp` ✓ | `playar-logo.webp` ✓ |
| Cal & Canto | `cal-canto.webp` ✓ | `cal-canto-logo.png` ✓ |
| RHEVO | `rhevo.webp` ✓ | `rhevo-logo.png` ✓ |

All logos are **white-on-transparent** — they sit on top of the dark gradient color block and remain visible over photos. The card thumbnail tries to load the concept image first; if unavailable the color-block placeholder shows instead. Both states display the logo centered in the thumb.

To replace a card photo with a higher-res version, just overwrite the matching `.webp` file — no HTML changes needed.

Recommended card photo size: **900×675px minimum**, 4:3 ratio, WebP format.

## Blog — Publishing a Post

1. Create a new file in `content/posts/` named `[slug].json` (slug = URL-friendly title, lowercase, hyphens)
2. Fill in the post JSON (see `content/posts/invertir-lotes-yucatan-2026.json` as reference)
3. Run: `node build.js`
4. Commit the updated `blog/` directory and `sitemap.xml`

### Post JSON fields

```json
{
  "slug":            "url-friendly-title",
  "title_es":        "Título en español",
  "title_en":        "Title in English",
  "description_es":  "SEO description ~155 chars (Spanish)",
  "description_en":  "SEO description ~155 chars (English)",
  "date":            "2026-05-26",
  "author":          "Agente RH",
  "category":        "Categoría",
  "category_en":     "Category",
  "tags":            ["tag1", "tag2"],
  "image":           "assets/images/blog-my-image.jpg",
  "image_alt_es":    "Alt text in Spanish",
  "image_alt_en":    "Alt text in English",
  "content_es":      "<p>HTML content in Spanish</p>",
  "content_en":      "<p>HTML content in English</p>"
}
```

## rh-agente Integration

Three integration points are pre-wired but commented out. Uncomment the one you activate first.

### 1. WhatsApp CTA (floating button + inline buttons)
In `js/main.js` — uncomment the `initWhatsAppFloat()` block and its CSS comment.
All "Cotizar" buttons already have a commented `href` pointing to `+52 999 370 8117`.

### 2. Embedded Chat Widget
In `index.html` → `#contacto` section — uncomment the `<div id="rh-chat-widget">` block.
In `js/main.js` — uncomment `initChatWidget()` and set `AGENT_WIDGET_URL`.

### 3. Contact Form → CRM Webhook
In `index.html` → `#contacto` section — uncomment the `<form id="lead-form">` block.
In `js/main.js` — uncomment `initLeadForm()` and set `CRM_WEBHOOK_URL`.

## Deploying

This is a pure static site — no build step required for the main site. Just serve the `joelgruporh/` folder from any static host (Netlify, Vercel, GitHub Pages, Cloudflare Pages, Apache/Nginx).

For clean blog URLs (`/blog/gran-verona/` instead of `/blog/gran-verona/index.html`) to work without explicit `index.html`, most hosts handle directory indexes automatically. If using Nginx, add:

```nginx
try_files $uri $uri/ $uri/index.html =404;
```
